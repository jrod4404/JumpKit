### 🔒 Secure Team Join Model — Full Design

**Current state (insecure):**
- Join only checks: org name + team name + password hash
- Anyone with the invite email forwarded to them can join

**Target state (3-factor join):**
1. Correct org name + team name
2. Correct team password (PBKDF2 hash match)
3. User's Supabase account email must exist as a `pending` invite in `team_invites` for that team

**Remaining attack vector + fix:**
- Attack: Mallory creates JumpKit account using Alice's email before Alice does, then joins
- Fix: Enable Supabase email confirmation — new accounts must verify email via inbox link before active
- Since only Alice can access her inbox, Mallory is blocked

**Implementation steps:**
1. **Add invite check to `joinTeam()` in `teams.js`:**
   - Query `team_invites` where `team_id = X AND email = currentUserEmail AND status = 'pending'`
   - If no matching invite → reject with "You haven't been invited to this team"
   - If found → proceed, update invite status to `'accepted'`
2. **Enable Supabase email confirmation:**
   - Supabase Dashboard → Auth → Email → "Confirm email" ON
   - Resend already configured as email provider
   - Must solve redirect URL for Electron (same challenge as forgot password)
3. **Electron deep-link solution** for both email confirmation + forgot password redirects
