## Windows Build Lessons Learned

### Final working test build config (`electron-builder.test.json`)
- `productName: "JumpKit"` — must be explicit, otherwise npm `name` ("jumpkit") is used for installer title
- `executableName: "jumpkit-test"` — avoids NSIS "cannot be closed" conflict on reinstall
- `extraMetadata: { jkTestBuild: true }` — signals test build; skips admin guard in main.js
- `nsis.oneClick: false` — wizard installer, no forced close behavior
- `nsis.include: "build/installer.nsh"` — overrides `customCheckAppRunning` macro as no-op
- `win.icon: "assets/icon.ico"` — must be ≥256x256; generated from icon_gradient_512.png via Python/Pillow

### Root causes of "jumpkit cannot be closed" error
1. Old JumpKit install at a different path (second install found/removed)
2. `jumpkit-updater` folder left by Squirrel updater
3. Stale NSIS registry entries
4. **Real fix**: rename executable to `jumpkit-test` so NSIS never finds a conflict

### Admin guard stall fix
- `app.isPackaged && !_pkg.jkTestBuild` — test builds skip the admin file guard
- Without this, the guard fires a blocking `dialog.showErrorBoxSync` that renders hidden on Windows

### ICO generation (macOS → Windows)
- `assets/icon.ico` was 16x16 — too small for electron-builder (needs ≥256x256)
- Generated with Python/Pillow: multi-size ICO (16/32/48/64/128/256px) from `icon_gradient_512.png`
