### 🔒 Secure Team Join Model — Design & Next Steps

**Current state (insecure):**
- Join only checks: org name + team name + password hash
- Anyone with the invite email can join — no email validation

**Target state (secure — 3-factor join):**
1. Correct org name + team name
2. Correct team password (PBKDF2 hash match)
3. **User's Supabase email must exist as a pending invite in `team_invites` for that team**

**Remaining attack vector + fix:**
- Attack: Mallory creates JumpKit account with Alice's email before Alice does
- Fix: Enable **Supabase email confirmation** — account not active until user clicks verification link in their inbox. Since only Alice can access her inbox, Mallory is blocked.

**Implementation steps:**
1. **Enable Supabase email confirmation** in Supabase Dashboard → Auth → Email → "Confirm email" toggle ON
   - Resend is already configured as the email provider
   - Must configure redirect URL for Electron (same challenge as forgot password flow)
2. **Add invite email check to join flow** in `teams.js` `joinTeam()` function:
   - Before inserting into `team_members`, query `team_invites` where `team_id = X AND email = currentUserEmail AND status = 'pending'`
   - If no matching invite → reject with "You haven't been invited to this team"
   - If found → proceed with join, then update invite status to `'accepted'`
3. **Test the full flow** end to end

**Files to change:**
- `teams.js` → `joinTeam()` function — add invite check before `team_members` insert
- Supabase Dashboard → Auth settings — enable email confirmation
- Possibly `index.html` / `auth.js` — handle email confirmation deep-link in Electron

**Note:** Supabase email confirmation redirect URL is the same problem as the forgot password flow — Electron can't receive standard web redirects. Need to design a custom solution (e.g. custom protocol handler or web-based intermediate page).
