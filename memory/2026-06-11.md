### Key Technical Decisions
- All home page click handlers use `data-jaction` attribute routing (CSP-compliant, no inline `onclick`)
- Switch cases added: `nav-stats`, `nav-teams`, `nav-account`, `nav-help`, `open-feedback-modal`
- Teams section: no `org_id` dependency — uses `supabaseClient.auth.getSession()` directly, queries by `owner_id`; RLS handles access
- Team member counts: batched Supabase queries, not N+1
- Home page grids: `repeat(4, 1fr)` default, breakpoints at ≤900px (2 col) and ≤500px (1 col)
- Consistent hover: `translateY(-2px)` + box-shadow on all stat/team cards matching tip-card style
- Section spacing: uniform via CSS `:not(:first-child)` margin-top

### Supabase SQL (for reference)
SQL to reset `sales@jumpkit.app` from cancelled → free:
```sql
UPDATE profiles
SET subscription_tier = 'free', subscription_status = 'free', trial_launches_used = 0
WHERE id = (SELECT id FROM auth.users WHERE email = 'sales@jumpkit.app');
```
