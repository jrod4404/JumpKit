## Windows SmartScreen / Code Signing (2026-06-11 evening)
- Jeff noticed "software is from an unknown developer" warning when installing JumpKit on Windows
- Root cause: Windows Defender SmartScreen triggers without a trusted Authenticode cert (or with a new cert lacking reputation)
- Options:
  - OV Code Signing Cert (~$100-180/yr, Sectigo) ‚Äî signs with signtool.exe, reputation builds over time
  - EV Code Signing Cert (~$300-500/yr, DigiCert/Sectigo) ‚Äî full SmartScreen trust immediately, needs hardware token

- Supabase sends the reset email anyway + auto-confirms email when reset link is used
- Behavior is safe and intentional ‚Äî no fix needed

**User pays for Unlimited BEFORE creating an account:**
- Root cause: `ls-webhook` does `.update().eq('email')` ‚Äî silently updates 0 rows if no profile exists
- Fix implemented:
  - `pending_upgrades` table created in Supabase (email, tier, ls_customer_id)
  - `ls-webhook` updated: detects 0-row update ‚Üí upserts to `pending_upgrades`, skips welcome email
  - New `apply-pending-upgrade` edge function (deployed, ACTIVE): checks pending_upgrades on login, applies upgrade, deletes row, sends welcome-core email
  - `auth.js` updated: fires `apply-pending-upgrade` fire-and-forget after every successful login
- Migration file: `app/supabase/migrations/20260611_pending_upgrades.sql`
