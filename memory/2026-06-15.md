## Auto-Backup Confirmation Modal (new feature)

### Problem
Auto-backup was firing the OS save dialog immediately on login with no warning.

### Solution
Added in-app confirmation modal before `saveBackup` IPC call in `runCloudBackup()`:
- Title: "Auto-Backup Ready" with database-export icon
- Body: "Your JumpKit data is ready to back up. Save a copy to your chosen location now?"
- [Save Backup] → OS save dialog
- [Skip] → toast "Auto-backup skipped." (deferred 1s via setTimeout)
- OS dialog cancel → no notification (result?.canceled check)

### Toast timing bug
- `runCloudBackup()` called from `initApp()` at line 171
- `const Toast` defined at line 943 — temporal dead zone during init
- Fix: `setTimeout(() => window.Toast?.success(...), 1000)` defers until after init completes
- Commits: `83ce9e2`, `d9a1c12`, `cd329fa`, `3651a14`
