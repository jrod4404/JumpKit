## X OAuth 2.0 + Auto-Publishing (15:03–15:14 EDT)

### What was built
- **`x-oauth.js`** module — full OAuth 2.0 PKCE flow for X (Twitter)
  - `GET /api/x/connect` → generates code_verifier/challenge, redirects to X authorize URL
  - `GET /oauth/x/callback` → exchanges code for access+refresh tokens, stores in settings table
  - Token storage: `x.access_token`, `x.refresh_token`, `x.connected` in settings
  - Auto-refresh on expired tokens
- **Publishing endpoint**: `POST /api/x/publish`
  - Posts text + up to 4 images via X API v2 + v1.1 media upload
  - Media upload uses chunked INIT/APPEND/FINALIZE flow
- **Scheduler worker**: checks every 60s for scheduled X posts that are due → auto-publishes
  - Started via `xOAuth.startSchedulerWorker(db)` in `server.listen` callback
- **PostKit UI**:
  - Settings → Connected Accounts section with X Client ID, Client Secret fields + Connect/Disconnect button
  - Post cards on X tab → "Publish Now" green button when post is scheduled + X connected
  - Status display shows ✓ Connected or ⚠ Not Connected
