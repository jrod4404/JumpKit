## Team ROI Feature — Implemented (afternoon)

### Decision
- Free tier: Option A — user's own shared jump stats from local SQLite + estimated team total with upgrade CTA
- Unlimited tier: Option B — true per-member aggregate stats synced to Supabase `member_stats` table

### Files changed
- `supabase/schema.sql` — added `member_stats` table + 2 RLS policies
- `supabase/migrations/20260609_member_stats.sql` — new migration file
- `js/sync.js` — added `syncMemberStats()` function, hooked into `syncSharedJumps()` (unlimited users only)
- `js/app.js` — added `renderTeamROISection()` function, wired into `renderStatsDash()` + `exportStatsPDF()`

### member_stats table schema
- `user_id` UUID FK → profiles
- `team_id` UUID FK → teams
- `total_launches` INTEGER
- `total_seconds_saved` INTEGER
- `dollars_per_hour` DECIMAL(10,2)
- `updated_at` TIMESTAMPTZ
- UNIQUE(user_id, team_id)
- RLS: `member_stats_self` (member reads/writes own row), `member_stats_owner_read` (team owner reads all)

### Migration SQL
Jeff ran this in Supabase SQL Editor to create the table.

### Behavior
- Stats page Summary view shows Team ROI section below personal stats
- Free: "Your Contribution" card (real data) + grayed "Estimated Team Total" + upgrade CTA
- Unlimited: Per-member table with launches, time saved, $ saved, last sync
- PDF: Team ROI section appended after personal summary (unlimited only)
- Sync fires on every `syncSharedJumps()` call for unlimited users
