# 2026-05-21

## JumpKit localStorage Security Audit & Cleanup

Full audit of all localStorage keys. Identified 10 keys total.

### Decisions made:
- **Items 1-3** (jk_theme, jk_sidebar_collapsed, jk_teams_collapsed): Keep in localStorage — UI state only, harmless.
- **Items 4-6** (jk_subscription_status, jk_subscription_tier, jk_role): **REMOVED** — security risk, could be tampered to bypass paywalls/roles. Now read exclusively from `window._supabaseProfile` (in-memory, resets from Supabase on every load). Tamper-proof.
- **Item 7** (jk_current_user): **REMOVED** — redundant with Supabase session. `DB.setSession/clearSession` made no-ops. Offline fallback simplified to redirect-to-login.
- **Item 8** (jk_seeded_<userId>): **Migrated to Supabase** — new `seeded_at` column on `profiles` table. Migration `20260521_add_seeded_at.sql` written and already executed in Supabase dashboard.
- **Item 9** (jk_notifs_<userId>): Keep for now — notifications are device-local events (auto-archive, backup). Revisit when team/cross-device notifications are built.
- **Item 10** (jk_last_sync): **DELETED** — was written but never read anywhere. Dead code.

### Files changed:
- `js/app.js` — removed 3 localStorage.setItem writes for subscription/role, updated cleanup sweep, simplified auth fallback
- `js/db.js` — removed localStorage reads for subscription/role in prefs, seed check now uses Supabase profiles.seeded_at
- `js/jumps.js` — removed localStorage fallbacks for subscription_tier (x2)
- `js/teams.js` — removed localStorage fallbacks for subscription_tier
- `js/sync.js` — removed dead jk_last_sync write
- `js/tests.js` — updated Test 4 and Test 36 to check window._supabaseProfile instead of localStorage
- `supabase/schema.sql` — added seeded_at column to profiles
- `supabase/migrations/20260521_add_seeded_at.sql` — new migration file (already applied)
- `supabase/migrations/README.md` — updated

### Final localStorage state (4 keys remain, all UI state):
| Key | Purpose |
|---|---|
| jk_theme | Dark/light mode |
| jk_sidebar_collapsed | Sidebar state |
| jk_teams_expanded | Team expand/collapse state (renamed from jk_teams_collapsed) |
| jk_notifs_<userId> | Local notification inbox |

---

## Teams Page — Default Collapsed Fix

- Teams were showing expanded by default due to old localStorage key `jk_teams_collapsed` using ambiguous format (true=collapsed, false=expanded)
- Renamed key to `jk_teams_expanded` with inverted semantics (only stores explicitly-expanded teams as true)
- Root cause of debug difficulty: Electron was serving cached JS (static ?v= param never changed). Fixed by adding `session.clearCache()` before `loadFile()` in dev mode in `main.js`.

---

## UI Changes

- **Settings merged into Account page** — 3rd tab added: My Account | My Teams | Settings. `renderSettings()` now redirects to `renderAccount('settings')`. Settings nav item removed from sidebar.
- **Sidebar nav labels removed** — "MAIN" and "INFO" section labels removed. "ADMIN" kept (hidden unless admin).
- **Home nav item** — 10px top margin added for spacing.
- **Backups**: `app_backup_2026-05-21`, `app_backup_2026-05-21b`
