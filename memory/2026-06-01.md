## App Bug Fixes (2026-06-01)

### Bug 1: Owner's shared cols rendering twice on login
- Root cause: `syncSharedJumps` fetched `shared_columns` for all teams user is a member of, including their own teams — creating duplicate old-format shadow columns
- Fix: query owned team IDs at start of sync, skip any remote cols belonging to owned teams
- Fix 2: one-time cleanup removes existing old-format shadow duplicates on login
- File: `app/js/sync.js`

### Bug 2: Renamed shared cols not updating on Teams page or Manage Sharing modal
- Root cause: `sharedTeams` array was NEVER persisted to SQLite (schema had no column for it) — only lived in memory cache
- After app restart, `sharedTeams` came back empty → rename sync skipped (no supabaseId) → Supabase name never updated → manage sharing modal showed cols as unshared
- Fix 1: Added `sharedTeams TEXT` column to SQLite schema via migration (`main.js`)
- Fix 2: `get-columns` IPC now deserializes JSON → array on read; includes one-time migration for old-format cols (isShared + teamId → sharedTeams[])
- Fix 3: `save-columns` and `save-column` IPC now serialize sharedTeams → JSON string on write
- Fix 4: Added `rebuildOwnerSharedTeams()` in sync.js — runs 4s after login, matches remote `shared_columns` to local cols missing sharedTeams (4-pass: supabaseId → name → order → sole pair), then pushes local name to Supabase where it differs
- Files: `app/main.js`, `app/js/sync.js`
- Confirmed working by Jeff ✅
