# 2026-07-07 - PostKit Session

## Major milestones today
- PostKit is fully working end-to-end with Auri automation
- Jeff tested the full flow and called it "amazing"

## What got built / fixed

### Platform Settings UI
- Added per-platform config cards (X, LinkedIn, YouTube)
- Settings: Posts/week, Posts/root, Best Posting Times
- Auri version shown as gray hint text (removed pill/icon)
- Save Settings persists to DB; fixed a bug where saves weren't sticking on refresh
- Posts/week and Posts/root are now separate controls (posts/root drives Auri generation count)

### Auri Auto-trigger
- Clicking "Send to Auri" now automatically spawns Auri via `hermes --profile auri chat -q "..."` in background
- No more manual Hermes step needed
- Command runs detached; PostKit responds instantly

### Status Polling
- PostKit polls every 8s while any seed is "processing"
- Status pill: processing (yellow) → "Content Generated" (blue) on completion  
- Toast fires: "Auri finished — posts ready!"
- Detail pane auto-refreshes when Auri is done

### Post Management
- Delete button (trash icon) added to each generated post
- DELETE /api/posts/:id endpoint was already in server, just wired frontend

### Posts/root Setting
- New DB field: platform.{x/linkedin/youtube}.posts_per_root
- Defaults: X=3, LI=1, YT=1
- Jeff set: X=10, LI=2, YT=1 for JumpKit content strategy
- Auri skill updated to use posts_per_root for generation count (not posts/week)

### LinkedIn Integration (FULL)
- linkedin-oauth.js built from scratch (OAuth 2.0, token exchange, refresh, ugcPosts publish)
- Server endpoints: /oauth/linkedin/callback, /api/linkedin/connect, /api/linkedin/publish, /api/linkedin/status, /api/linkedin/disconnect
- LinkedIn scheduler worker (auto-publishes due scheduled posts every 60s)
- Settings UI: Client ID/Secret fields + Connect/Disconnect button with green Connected pill
- "Publish Now" button on scheduled LinkedIn posts
- Required LinkedIn Developer Products: "Share on LinkedIn" + "Sign In with LinkedIn using OpenID Connect"
- Fixed routing bug: callback was inside /api/ guard, moved before it

### Platform Count Pills on Content Cards
- Each seed card now shows colored pills: X (charcoal), LinkedIn (blue 95%), YouTube (red 95%)
- Counts pulled from DB GROUP BY platform query added to getSeeds

### Backup
- PostKit-backup-2026-07-07 created in Topics/

## PostKit Subpage / Detail Pane Fixes (Evening session)

### Image preview bug — FIXED
- Images in the seed detail pane weren't enlarging on click
- Root cause: clicking image opened lightbox, but the lightbox overlay logic was broken
- Fixed: image click now properly shows enlarged view / lightbox

### Subpage always-visible bug — FIXED
- `display: flex !important` in CSS was overriding `style="display:none"` on subpage
- Fixed by removing `!important` and explicitly setting display in JS:
  - `openSeedDetail()` → sets `display: flex`
  - `closeSeedDetail()` → sets `display: none`
  - `switchTab()` also hides subpage correctly

### Back link added
- "‹ back to Content Page" link added to subpage (top left, accent color)

### Schedule button behavior (per Jeff's question ~19:05)
- The "Schedule" button on each generated post sets a scheduled_at datetime
- X scheduler worker auto-publishes due posts every 60s
- LinkedIn scheduler worker also checks every 60s
- If a post is scheduled, the button shows the scheduled time; clicking again lets you reschedule
