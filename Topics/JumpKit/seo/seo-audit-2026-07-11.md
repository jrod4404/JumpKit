## Executive Summary

JumpKit has a clean, fast-loading landing page with solid positioning ("Stop Searching. Start Jumping.") but is critically underequipped for organic search and AI visibility. The site has **only 1 indexed page** in search results, **zero structured data**, **no meta description on the homepage**, **no sitemap**, **no robots.txt**, and **no canonical tags**. These are foundational issues that prevent Google (and AI engines) from understanding, indexing, and ranking the site properly.

**Good news:** The product positioning is solid, the brand is clean, and the fixes are straightforward. With 4–8 weeks of focused work, JumpKit can establish a real organic presence.

### Top 3 Priority Issues
1. 🚨 **No meta description + no Open Graph tags** — homepage gets zero SERP enhancement
2. 🚨 **No sitemap.xml + no robots.txt** — Google has no crawl guidance
3. 🚨 **No structured data (Schema)** — no rich results possible; AI engines can't extract structured info

### Quick Wins
- Add meta description to homepage (30 min)
- Add OG/Twitter card tags (1 hr)
- Add canonical tags to all pages (1 hr)
- Create sitemap.xml and robots.txt (1 hr)
- Add JSON-LD Organization + SoftwareApplication schema (2 hrs)

### 1.2 Crawlability

| Check | Status | Notes |
|-------|--------|-------|
| robots.txt | 🔴 MISSING | Returns 404. Google has no crawl guidance |
| sitemap.xml | 🔴 MISSING | Returns 404. No sitemap submitted |

| llms.txt | 🔴 MISSING | AI engines get no context file |
| Crawl directives in HTML | 🔴 MISSING | No `<meta name="robots">` tag found |

**Critical Issue:** Without `robots.txt`, Google's crawler operates blind. Without `sitemap.xml`, Google may not discover all pages (currently only `jumpkit.app/` and `jumpkit.app/blog/introducing-jumpkit` are indexed). The blog page itself appears to have content but no linked posts from the index — this means blog posts may be undiscoverable.

**Fix:**
```
# /robots.txt
User-agent: *
Allow: /
Sitemap: https://jumpkit.app/sitemap.xml

# Allow AI crawlers explicitly
User-agent: GPTBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: Google-Extended
Allow: /
```

## Part 3: Schema & Structured Data

### 3.1 Current State

**JSON-LD Schema: NONE FOUND on any page.**

This is a significant gap. No rich results are possible (no star ratings, FAQ accordion, product pricing, app info, etc. in SERPs). AI engines also use schema to extract structured facts about your product.

### 3.2 Recommended Schema (Priority Order)

**Priority 1 — Organization (homepage `<head>`)**
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "JumpKit",
  "url": "https://jumpkit.app",
  "logo": "https://jumpkit.app/assets/logo/jumpkit-light.png",
  "sameAs": [
    "https://x.com/jumpkitapp",
    "https://youtube.com/@jumpkitapp",
    "https://linkedin.com/company/jumpkitapp"
  ],
  "contactPoint": {
    "@type": "ContactPoint",
    "contactType": "customer support",
    "email": "support@jumpkit.app"
  }
}
```

**Priority 2 — SoftwareApplication (homepage)**
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "JumpKit",
  "operatingSystem": "Windows, macOS",
  "applicationCategory": "ProductivityApplication",
  "offers": [
    {
      "@type": "Offer",
      "name": "JumpKit Free",
      "price": "0",
      "priceCurrency": "USD"
    },
    {
      "@type": "Offer",
      "name": "JumpKit Unlimited",
      "price": "10",
      "priceCurrency": "USD",
      "billingIncrement": "P1M"
    }
  ],
  "description": "JumpKit replaces scattered bookmarks with one organized, searchable, shareable launcher for web links, local folders, and team resources.",
  "url": "https://jumpkit.app"
}
```

**Priority 3 — FAQPage (add an FAQ section to homepage or a dedicated FAQ page)**
```json
{
  "@context": "https://schema.org",
```
