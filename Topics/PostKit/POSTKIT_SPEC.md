## 1. Overview

PostKit is a **local-first localhost web app** for streamlining social media content creation, scheduling, and tracking across **X (Twitter)**, **LinkedIn**, and **YouTube**.

It is a **human visualization and scheduling layer** — the AI work (content optimization and scheduling) is done by a separate agent called **Hermes** which writes directly to PostKit's SQLite database.

PostKit does **not** do AI itself. Its job is:
- Let humans create seed content packages (text + media)
- Queue that seed for Hermes to process (via SQLite job queue)
- Display and manage what Hermes generates
- Track what gets posted

## 4. Database Schema

### `seeds`
Seed content packages created by the human.

| Column | Type | Description |
|---|---|---|
| id | TEXT PK | UUID |
| title | TEXT | Seed title |
| body | TEXT | Main text content |
| tags | TEXT | JSON array of tag strings |
| campaign | TEXT | Optional campaign/series name |
| template | TEXT | Template name if used |
| status | TEXT | `draft` → `processing` → `done` / `error` |
| created_at | INTEGER | Unix ms timestamp |
| updated_at | INTEGER | Unix ms timestamp |

### `seed_media`
Media files attached to seed content.

| Column | Type | Description |
|---|---|---|
| id | TEXT PK | UUID |
| seed_id | TEXT FK | → seeds.id (CASCADE DELETE) |
| filename | TEXT | Saved filename in `media/` folder |
| original_name | TEXT | Original uploaded filename |
| mime_type | TEXT | e.g. `image/jpeg` |
| size | INTEGER | Bytes |
| created_at | INTEGER | Unix ms timestamp |

### `posts`
Platform-specific posts generated by Hermes from a seed.

| Column | Type | Description |
|---|---|---|
| id | TEXT PK | UUID |
| seed_id | TEXT FK | → seeds.id (SET NULL on delete) |
| platform | TEXT | `x` / `linkedin` / `youtube` |
| post_text | TEXT | Optimized post text |
| media_paths | TEXT | JSON array of filenames |
| scheduled_for | INTEGER | Unix ms timestamp |
| status | TEXT | `draft` / `scheduled` / `posted` |
| posted_at | INTEGER | When marked posted |
| version | INTEGER | Revision number |
| notes | TEXT | Human notes |
| created_at | INTEGER | Unix ms timestamp |
| updated_at | INTEGER | Unix ms timestamp |

### `analytics`
Manual performance tracking per post.

| Column | Type | Description |
|---|---|---|
| id | TEXT PK | UUID |
| post_id | TEXT FK | → posts.id (CASCADE DELETE) |
| likes | INTEGER | |
| comments | INTEGER | |
| views | INTEGER | |
| shares | INTEGER | |
| recorded_at | INTEGER | Unix ms timestamp |

### `jobs`
Hermes job queue. PostKit writes `pending` records; Hermes polls and processes them.

| Column | Type | Description |
|---|---|---|
| id | TEXT PK | UUID |
| seed_id | TEXT FK | → seeds.id (CASCADE DELETE) |
| status | TEXT | `pending` / `processing` / `done` / `error` |
| error | TEXT | Error message if failed |
| created_at | INTEGER | Unix ms timestamp |
| updated_at | INTEGER | Unix ms timestamp |

### `settings`
Key-value app config.

| key | default | Description |
|---|---|---|
| `app.port` | `8788` | Server port |
| `app.theme` | `dark` | UI theme |
| `hermes.config_path` | `""` | Path to Hermes agent config file |
| `platform.x.frequency` | `5` | Posts per week target |
| `platform.x.best_times` | `["9:00","12:00","17:00"]` | Optimal posting times |
| `platform.linkedin.frequency` | `3` | Posts per week target |
| `platform.linkedin.best_times` | `["8:00","12:00","17:30"]` | Optimal posting times |
| `platform.youtube.frequency` | `1` | Posts per week target |
| `platform.youtube.best_times` | `["15:00","18:00"]` | Optimal posting times |

## 5. REST API

### Seeds
| Method | Path | Description |
|---|---|---|
| GET | `/api/seeds` | List seeds (search, status, campaign filters) |
| POST | `/api/seeds` | Create new seed |
| GET | `/api/seeds/:id` | Get seed with posts, media, jobs |
| PUT | `/api/seeds/:id` | Update seed |
| DELETE | `/api/seeds/:id` | Delete seed (cascade) |
| POST | `/api/seeds/:id/finalize` | Finalize seed → creates pending job |
| POST | `/api/seeds/:id/media` | Upload media (base64 JSON body) |
| DELETE | `/api/seeds/:id/media/:mid` | Delete media file |

### Posts
| Method | Path | Description |
|---|---|---|
| GET | `/api/posts` | List posts (platform, status, seed_id, date filters) |
| GET | `/api/posts/today` | Today's queue (scheduled today, not posted) |
| PUT | `/api/posts/:id` | Update post (text, schedule, status, notes) |
| DELETE | `/api/posts/:id` | Delete post |

### Calendar
| Method | Path | Description |
|---|---|---|
| GET | `/api/calendar?from=&to=` | Posts in date range for calendar view |

### Analytics
| Method | Path | Description |
|---|---|---|
| GET | `/api/analytics` | All analytics with post/seed context |
| POST | `/api/analytics` | Create analytics record |
| PUT | `/api/analytics/:id` | Update analytics |

### Jobs
| Method | Path | Description |
|---|---|---|
| GET | `/api/jobs` | Recent jobs (last 50) |
| GET | `/api/jobs/pending` | Pending jobs for Hermes to poll (includes seed + media) |
| PUT | `/api/jobs/:id` | Update job status (Hermes calls this) |

### Settings & Hermes
| Method | Path | Description |
|---|---|---|
| GET | `/api/settings` | Get all settings |
| PUT | `/api/settings` | Update settings (key-value map) |
| GET | `/api/hermes/config` | Read Hermes config file at configured path (read-only) |

### Media
| Method | Path | Description |
|---|---|---|
| GET | `/media/:filename` | Serve media file |

## 6. Hermes Integration

### How it works
1. **Human finalizes** a seed in PostKit UI
2. PostKit writes a `jobs` record with `status = 'pending'`
3. **Hermes polls** `GET /api/jobs/pending` on its own schedule
4. Hermes runs its **dissect skill** on the seed content
5. Hermes writes platform-specific posts to the `posts` table directly in SQLite
6. Hermes schedules posts (sets `scheduled_for` timestamps)
7. Hermes calls `PUT /api/jobs/:id` with `status = 'done'`
8. PostKit auto-updates seed status to `done`

### Hermes Connection Setting
- Settings page has a **"Hermes Config Path"** field
- Enter the path to Hermes's config/skill JSON file
- PostKit reads it and displays the marketing skill settings **read-only**
- This lets humans confirm Hermes's strategy without leaving PostKit

## 7. UI Tabs

### Content
- Seed library (card grid) with search + filter (status, campaign)
- "New Seed" button → create form
- Click seed → detail view:
  - Seed body + media thumbnails
  - Status + job status
  - Generated posts by platform (post text, schedule, copy button, edit button)
  - "Finalize & Send to Hermes" button (when draft)

### Calendar
- Month / Week / Day views
- Color-coded post dots/chips by platform:
  - X = `#1d9bf0` (Twitter blue)
  - LinkedIn = `#0077b5` (LinkedIn blue)
  - YouTube = `#ff0000` (YouTube red)
- Click day → list of posts for that day
- Drag-to-reschedule (optional, Phase 2)

### Today
- Posts scheduled for today, not yet posted
- Grouped by platform
- Each post card: text preview, copy-to-clipboard button, "Mark as Posted" button
- Visual status badges

### Analytics
- Table/cards of posted content
- Manual metric input: likes, comments, views, shares
- Filterable by platform

### Settings
- **App Settings:** port, theme toggle
- **Hermes Connection:** config file path field + "Test Connection" button
- **Hermes Skill Settings (read-only):** JSON display of Hermes config
- **Platform Posting Criteria:** frequency targets, best times per platform

## 9. Development Notes

- Start server: `npm install && npm start`
- Media uploads: base64-encoded in JSON body (no multipart)
- All timestamps: Unix milliseconds
- SQLite WAL mode + foreign keys enabled
- Better-sqlite3 is synchronous — no async/await needed in API handlers
